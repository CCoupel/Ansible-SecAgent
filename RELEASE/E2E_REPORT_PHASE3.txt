INTEGRATION FLOW VALIDATION
========================================================

Flow: Ansible ↔ Plugins ↔ Server ↔ Agent

Step 1: Inventory Discovery
  1. Ansible invokes inventory_plugins/secagent_inventory.py
  2. Plugin reads RELAY_TOKEN_FILE env var
  3. Plugin calls GET /api/inventory with JWT (role: "plugin")
  4. Server queries agents table → {qualif-host-01: connected}
  5. Returns Ansible format:
     {
       "all": {"hosts": ["qualif-host-01"]},
       "_meta": {
         "hostvars": {
           "qualif-host-01": {
             "ansible_connection": "relay",
             "secagent_status": "connected"
           }
         }
       }
     }
  6. ✓ VERIFIED: Inventory plugin can fetch agents

Step 2: Command Execution
  1. Ansible uses connection_plugins/secagent.py for host
  2. Plugin loads JWT from RELAY_TOKEN_FILE
  3. Plugin calls POST /api/exec with {hostname, command, stdin}
  4. Server receives task → forwards via NATS to agent
  5. Agent executes via subprocess
  6. Server returns {rc, stdout, stderr}
  7. Ansible processes response
  8. ✓ FLOW TESTED: All components respond correctly

Step 3: File Transfer
  1. Ansible needs put_file() to copy module to agent
  2. Connection plugin reads local file
  3. Plugin base64 encodes (< 500KB)
  4. Plugin calls POST /api/upload
  5. Server forwards to agent
  6. Agent writes file with chmod
  7. Agent returns {rc}
  8. ✓ TESTED: File transfer endpoints accessible

Step 4: Result Retrieval
  1. Ansible needs fetch_file() to get results
  2. Connection plugin calls POST /api/fetch
  3. Server fetches from agent
  4. Agent base64 encodes file
  5. Server returns {rc, data}
  6. Plugin base64 decodes
  7. Ansible writes local file
  8. ✓ TESTED: Fetch endpoints accessible
